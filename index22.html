// Kompatibilität

Was AuditFlow analysieren kann

Transparenz über Möglichkeiten und Grenzen — damit du weißt, was funktioniert.

Funktioniert ✅

  • • Normale Websites (WordPress, Squarespace, Wix)
  • • E-Commerce Shops (Shopify, WooCommerce)
  • • SaaS-Produkte (öffentliche Seiten)
  • • Blogs, Magazine & Unternehmenswebsites
  • • Portfolio-Seiten & Landing Pages
  • • GitHub Pages, Vercel, Netlify Deployments
  • • Alles mit öffentlicher URL

Funktioniert nicht ❌

  • • Große Plattformen (Google, Amazon, Facebook, Instagram)
  • • Passwortgeschützte Seiten
  • • Intranet & lokale URLs (localhost, 192.168.x.x)
  • • Seiten hinter VPN
  • • Staging mit Basic Auth
  • • CORS-blockierte Domains

Der CORS-Workaround: HTML-Fallback

Wenn eine Seite CORS blockiert, kannst du einfach den HTML-Code kopieren und ins Dashboard einfügen. Das löst das Problem für ~80% der Fälle:

  1. Öffne die Seite im Browser
  2. Rechtsklick → "Seitenquelltext anzeigen"
  3. Kopiere den HTML-Code
  4. Füge ihn in AuditFlow ein
  5. Audit läuft sofort — ohne CORS-Problem!
